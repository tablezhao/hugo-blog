# 网站首页文章列表优化计划

## 优化目标
1. 将首页文章列表从仅显示featured文章调整为显示全部文章，每页10篇
2. 保持首页作者信息模块不变
3. 确保首页文章列表样式与文章页面保持视觉一致性
4. 优化分页功能，确保在主流浏览器中正常工作
5. 保证首屏加载时间不超过2秒

## 具体修改方案

### 1. 修改首页布局文件 (home.html)
- 将文章列表数据源从`where .Site.RegularPages "Params.featured" true`改为`.Site.RegularPages`
- 保持现有的分页设置（每页10篇）

### 2. 优化文章列表样式
- 修改`featured.html`部分，将文章列表样式从`.featured-single`改为与文章页面一致的`.blog-card`样式
- 将文章标题从h4改为h3，保持与文章页面一致
- 调整布局结构，使用与文章页面相同的`.blog-wrapper`和`.blog-list`容器

### 3. 增强分页功能
- 保持现有的上一页/下一页按钮
- 确保分页逻辑正确，包括页码导航、当前页码高亮显示
- 优化分页样式，确保在主流浏览器中正常工作

### 4. 保持作者信息模块不变
- 确保作者名称、作者描述、作者头像等功能和显示效果完全不受影响

## 实施步骤
1. 复制`blog/list.html`中的文章列表样式到`home.html`中
2. 修改`home.html`的文章数据源，显示全部文章
3. 调整`featured.html`或创建新的部分，实现与文章页面一致的样式
4. 测试分页功能在不同浏览器中的表现
5. 进行性能测试，确保首屏加载时间不超过2秒

## 预期效果
- 首页显示全部文章，每页10篇
- 文章列表样式与文章页面保持一致
- 分页功能正常，支持上一页/下一页导航
- 作者信息模块保持不变
- 首屏加载时间不超过2秒
- 兼容主流浏览器
